# 数据空间详细页面 —设计文档

## 一、页面概述

- **数据空间详细页面**主要对数据进行管理，为用户提供数据资产为用户提供数据资产的精细化管理、检索、筛选及数据登记能力

- 页面以**多维度分类 + 分步式操作**为核心设计逻辑，采用「左侧侧边导航 + 中间筛选条件区域 + 右侧数据资产展示区 + 右侧弹窗操作」的四区域布局，适配企业级数据管理的精细化、专业化需求，支持**数据集**、**数据API**、**高质量训练集**三类核心数据资产的统一管理。

- **数据空间详细页面**为SpaceDataList.vue，生成位置（DAM-vue/src/views/DAM/DataInfo/Space/SpaceData/SpaceDataList.vue）

## 二、页面跳转逻辑

### 2.1 跳转入口

从**数据空间管理页面**触发跳转，操作规则如下：

- 触发方式：**双击**数据空间管理页面中的任意数据空间卡片（如 “客户数据中心”“AI 训练数据集” 等）。
- 跳转参数：携带当前数据空间的**唯一标识（ID）**、**空间名称**、**空间类型（部门 / 项目 / 私有 / 公共）**、**创建人 / 成员信息**、**部门名称**、**部门ID**等核心参数，确保详细页面与原数据空间数据强关联。
- 返回：点击按钮可返回，当时选择部门的数据空间管理页面

## 三、整体布局架构

页面整体采用**三栏式主布局**，辅以右侧弹窗层，具体区域划分如下：

表格

| 区域位置    | 区域名称      | 核心功能                                                                                 | 布局占比（参考）  |
| ------- | --------- | ------------------------------------------------------------------------------------ | --------- |
| 左侧（固定）  | 数据类型侧边导航栏 | 展示核心数据资产分类                                                                           | 15%       |
| 中间（固定）  | 中间筛选条件区域  | 跟随左侧侧边栏选择变化，展示不同的细分筛选                                                                | 20%       |
| 右侧（主内容） | 数据资产展示区   | 主要展示符合筛选条件的数据集 / 数据 API / 高质量训练集卡片，支持检索、排序。同时在空间上方设置不同的“登记按钮”，分别是首次登记、转让登记、变更登记和注销登记 | 55%       |
| 最右侧（弹窗） | 数据登记操作弹窗  | 不同的登记按钮，右侧弹窗展示不同的页面内容                                                                | 50%（弹窗覆盖） |

## 四、左侧：数据类型侧边导航栏

### 4.1 设计风格

在标明数据类型导航栏
采用**浅色系背景 + 图标 + 文字**的极简风格，支持菜单展开 / 收起、选中态高亮，适配多分类层级的导航需求，视觉清晰且操作高效。

### 4.2 页面布局—侧边导航栏

| 区域位置 | 区域名称   | 核心功能                                                                           | 布局占比    |
| ---- | ------ | ------------------------------------------------------------------------------ | ------- |
| 上方   | 空间信息展示 | 包括当前数据空间的**唯一标识（ID）**、**空间名称**、**空间类型（部门 / 项目 / 私有 / 公共）**、**创建人 / 成员信息**等核心参数 | 10%-15% |
| 下方   | 导航菜单层级 | 详细参考4.3导航菜单层级                                                                  |         |

### 4.3导航菜单层级

导航栏为**一级分类**结构，包含企业级数据管理的核心资产类型，具体如下：

1. **数据集**（默认选中 / 首次加载展示）
   
   - 图标：📊（数据表格类图标）
   - 描述：存储结构化 / 非结构化数据资产的核心集合，是数据空间的基础资产类型。

2. **数据 API**
   
   - 图标：🔗（接口连接类图标）
   - 描述：封装数据资产的标准化接口，支持外部系统调用与数据交互。

3. **高质量训练集**
   
   - 图标：⚡（高性能 / 训练类图标）
   - 描述：经过清洗、标注、优化的优质数据资产，专用于 AI 模型训练场景。

### 4.3 交互规则

- 选中态：点击任意一级菜单，菜单文字与图标**高亮（主色调）**，同时中间功能菜单区切换为对应分类的子菜单。
- 展开 / 收起：支持侧边栏整体展开 / 收起（收起后仅显示图标），适配不同屏幕尺寸的适配需求。

跟随左侧数据类型侧边栏的选择，动态展示**数据类型的筛选条件**，是实现 “精准检索数据资产” 的核心交互区。菜单结构为**二级联动**，支持多级筛选条件的组合配置。

### 6.2 页面顶部操作栏

展示区顶部固定操作栏，包含以下功能：

## 九、视觉与交互规范

### 9.1 视觉规范

- 主色调：企业蓝（#1677FF），适配数据管理平台的专业、稳重属性。
- 辅助色：成功绿（#52C41A）、警告橙（#FAAD14）、错误红（#FF4D4F）、浅灰（#F5F7FA）。
- 字体：中文采用「思源黑体」，英文采用「Inter」，层级清晰（标题 18 号 / 正文 14 号 / 辅助 12 号）。
- 圆角：卡片 / 按钮 / 弹窗统一采用 8px 圆角，保持视觉一致性。

### 9.2 交互规范

- 所有可操作元素（按钮、标签、卡片）均需有 hover/active/ 选中态反馈。
- 长列表数据集支持**懒加载**，滚动到底部自动加载更多数据。
- 筛选条件变更后，右侧内容区需有**加载动画**，提升用户感知。
- 弹窗操作需有**过渡动画**（滑入 / 滑出），避免页面跳转的生硬感。
