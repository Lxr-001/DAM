# 数据资产审批管理 - 完整需求文档

---

## 一、页面概述

本功能包含两个核心页面：

1. **审批管理** - 审批者对待审批申请进行审批操作
2. **我的申请** - 申请人查看自己提交的申请及审批结果

---

## 二、页面结构

| 页面   | 路由                  | 访问角色 | 说明               |
| ---- | ------------------- | ---- | ---------------- |
| 审批管理 | /AssetApprovalList  | 审批者  | 待审批列表、审批操作、已审批历史 |
| 我的申请 | /MyAssetApplication | 申请人  | 申请记录、审批结果查看      |

---

## 三、审批管理页面

### 3.1 Tab页结构

- **待审批(数量)**：显示当前需要审批的申请数量
- **已审批(数量)**：显示本人已审批过的申请数量

### 3.2 筛选条件

| 条件   | 控件类型    | 说明                  |
| ---- | ------- | ------------------- |
| 数据空间 | 下拉选择    | 可选择当前用户有权限的数据空间     |
| 登记类型 | 下拉选择    | 首次登记/转让登记/变更登记/注销登记 |
| 提交时间 | 日期范围选择器 | 筛选提交时间区间            |

### 3.3 待审批列表字段

| 字段     | 说明                  |
| ------ | ------------------- |
| 序号     | 自动编号                |
| 资产名称   | 资产的主名称              |
| 登记类型   | 首次登记/转让登记/变更登记/注销登记 |
| 申请人    | 提交申请的用户姓名           |
| 所属数据空间 | 资产所在的数据空间名称         |
| 提交时间   | 申请提交的日期时间           |
| 操作     | 查看详情、审批通过、审批驳回      |

### 3.4 已审批历史记录列表字段

| 字段     | 说明                  |
| ------ | ------------------- |
| 序号     | 自动编号                |
| 资产名称   | 资产的主名称              |
| 登记类型   | 首次登记/转让登记/变更登记/注销登记 |
| 申请人    | 提交申请的用户姓名           |
| 所属数据空间 | 资产所在的数据空间名称         |
| 审批结果   | 通过/驳回               |
| 审批时间   | 本人审批的时间             |
| 操作     | 查看详情                |

### 3.5 审批操作

- **审批通过**：点击后直接通过，申请状态变为 `approved`，资产正式生效
- **审批驳回**：点击后弹出对话框，需填写驳回原因（必填），确认后申请状态变为 `rejected`

### 3.6 驳回原因弹窗

```
+------------------------------------------+
|  审批驳回                                 |
+------------------------------------------+
|  驳回原因：                               |
|  ┌────────────────────────────────────┐  |
|  │ 请输入驳回原因（必填）               │  |
|  └────────────────────────────────────┘  |
|                                          |
|         [取消]            [确认驳回]     |
+------------------------------------------+
```

## 四、我的申请页面

### 4.1 Tab页结构

- 全部
- 待审批
- 已通过
- 已驳回

### 4.2 筛选条件

与审批管理页面相同（数据空间、登记类型、提交时间）

### 4.3 列表字段

| 字段     | 说明                  |
| ------ | ------------------- |
| 序号     | 自动编号                |
| 资产名称   | 资产的主名称              |
| 登记类型   | 首次登记/转让登记/变更登记/注销登记 |
| 所属数据空间 | 资产所在的数据空间名称         |
| 提交时间   | 申请提交的日期时间           |
| 审批状态   | 待审批/已通过/已驳回         |
| 审批时间   | 审批的时间（仅已审批时显示）      |
| 操作     | 查看详情                |

### 4.4 查看详情

点击"查看详情"弹出抽屉，展示：

- 基本信息（资产名称、类型、数据空间、申请人等）
- 资产详情（根据登记类型不同展示不同字段）
- 审批记录（审批人、审批时间、审批结果、驳回原因）

## 五、审批结果通知机制

### 5.1 顶部导航栏消息提示

- 审批完成后，在系统顶部导航栏显示消息提示（小红点/数字角标）
- 点击消息图标展开下拉列表，显示最新审批结果
- 消息内容包含：资产名称、审批结果（通过/驳回）、审批时间

### 5.2 消息展示样式

```
+----------------------------------+
|  🔔 消息中心(3)              [×]  |
+----------------------------------+
|  ● 您的"XX资产"申请已审批通过   |

|    2026-04-22 16:45             |
```

## 六、登记类型说明

| 类型   | 英文标识     | 说明        |
| ---- | -------- | --------- |
| 首次登记 | initial  | 新资产入驻数据空间 |
| 转让登记 | transfer | 资产Owner变更 |

## 七、审批状态流转

```
申请人提交 → pending（待审批）
                ↓
        审批人审批
                ↓
    ┌───────────┴───────────┐
    ↓                       ↓
approved（通过）      rejected（驳回）
    ↓                       ↓
资产正式生效         驳回原因记录，申请人可查看
```

## 十二、需新增的数据库表

### sys_message（系统消息表）

| 字段          | 类型       | 说明                     |
| ----------- | -------- | ---------------------- |
| id          | VARCHAR  | 主键                     |
| user_id     | VARCHAR  | 接收消息的用户ID              |
| title       | VARCHAR  | 消息标题                   |
| content     | VARCHAR  | 消息内容                   |
| type        | VARCHAR  | 消息类型（approval_result等） |
| related_id  | VARCHAR  | 关联ID（如申请ID）            |
| is_read     | TINYINT  | 是否已读（0-未读，1-已读）        |
| create_time | DATETIME | 创建时间                   |
